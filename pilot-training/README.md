# /pilot-training

Pilot training materials for physicians, clinical staff, and operators.

Suggested contents:

- `physician-onboarding.md` — 10-minute walkthrough deck (link or markdown).
- `patient-comms-cards.md` — printable cards summarizing `PILOT_USER_GUIDE_PATIENT.md` for hand-off to patients.
- `operator-checklist.md` — daily operator checklist (smoke run, evidence sampling, incident log).
- `dry-run-script.md` — script for a dry-run encounter before live patient use.

Until populated, treat the following as required onboarding:

1. Read `pilot-package/PILOT_READINESS_MASTER.md`.
2. Read the role-appropriate user guide:
   - Physicians → `pilot-package/PILOT_USER_GUIDE_PHYSICIAN.md`.
   - Legal / Compliance → `pilot-package/PILOT_USER_GUIDE_LEGAL_COMPLIANCE.md`.
   - Operators / Engineering → `pilot-sops/SOP_SECURE_SIGNING.md`.
3. Review `pilot-scenarios/PILOT_TEST_SCENARIOS.md` and run S1 (Successful Signing) and S2 (Refusal Flow) end-to-end in a controlled setting before live use.
4. Confirm familiarity with `pilot-package/PILOT_FEEDBACK_TEMPLATE.md`.
