IMC REAL PILOT - HUMAN-IN-THE-LOOP (ER)

Scope
- Department: ER only
- Roles: Doctor (ER), Legal Admin
- Volume: 10 real refusal cases
- Workflow rule: No cross-role automation; each role must perform its own step in production

Role Assignment
- Doctor (ER): Create refusal case and complete clinical/refusal inputs
- Legal Admin: Review case, generate evidence bundle, click Verify, record outcome

Execution Sequence (per case)
1) Doctor login at https://wathiqcare.online/login
2) Doctor creates refusal case in ER workflow
3) Doctor records case id in tracker and marks handoff to Legal Admin
4) Legal Admin opens same case id
5) Legal Admin reviews case
6) Legal Admin opens /documents
7) Legal Admin generates bundle for that case id
8) Legal Admin clicks Verify for that bundle
9) Legal Admin records PASS/FAIL and elapsed time
10) Both roles submit quick feedback for that case

Timing Capture Rules
- Start time: Doctor clicks create case submit
- End time: Legal Admin gets Verify result (PASS or FAIL)
- Record total minutes per case in tracker

PASS Criteria (per case)
- Case created by Doctor
- Bundle generated by Legal Admin
- Verify result is PASS
- No blocker recorded

Immediate Escalation Triggers
- Login/session failure
- Doctor cannot create ER refusal case
- Legal Admin cannot access case/documents
- Bundle generation fails
- Verify returns FAIL

Required Logs
- Case tracker file: one row per case id
- Feedback file: one row per user per case
